# Synthetic Context Attribution Diagnostics

This document describes the diagnostic system used to evaluate and select synthetic context attribution examples. It provides formal definitions for **Syntactic Bias** and **Semantic Bias**, explains the mathematical formulations of the five diagnostic rules, and explains how the consolidated quality score is computed.

---

## 🧠 Core Concept: Dual Value Functions

To benchmark context attribution algorithms, we evaluate synthetic questions under two fundamentally different characteristic value functions:

1.  **Unweighted Log-Likelihood (ULL)**:
    - **Formulation**: $v(S) = LL(R \mid Q, S) - LL(R \mid Q, \emptyset)$
    - **Mechanism**: Evaluates the change in the model's logits for the exact tokens of the target response $R$. It is computed via teacher-forcing (no text generation).
    - **Limitation**: It is highly sensitive to token-level correlation (copying and lexical priming) rather than high-level semantic reasoning.

2.  **BERT Similarity (BERT)**:
    - **Formulation**: $v(S) = \text{sim}_{\text{BERT}}(R_{\text{grand}}, R_S) - \text{sim}_{\text{BERT}}(R_{\text{grand}}, R_{\emptyset})$
    - **Mechanism**: Evaluates the semantic similarity (using BERT/MPNet embeddings) between the response generated by the model under subset $S$ ($R_S$) and the response generated under the grand coalition of all context sentences ($R_{\text{grand}}$).
    - **Limitation**: It measures abstract semantic closeness and is easily fooled by sentences that describe the right event or context but lack the specific identifying entity.

---

## ⚖️ Definitions: Syntactic Copy-Bias vs. Semantic Bias

By contrasting ULL and BERT evaluations across carefully designed distractors, we isolate and measure two major categories of bias in LLM context attribution:

### 1. Syntactic Copy-Bias (Lexical Priming Bias)
*   **Definition**: The tendency of next-token-likelihood attribution metrics (like ULL) to assign disproportionately high importance to context sentences that share surface-level lexical tokens, exact keywords, or entity names with the target response, regardless of whether those sentences are semantically relevant or logically necessary for reasoning.
*   **How it Manifestes**: Language models act as associative retrieval engines. When a context sentence contains a rare target entity (e.g., "carburetor"), the probability of generating that word in the response is heavily primed. A likelihood-based method sees a massive spike in probability and incorrectly attributes high importance to the sentence, even if it is a complete distractor (e.g., "The carburetor caused the exhaust to backfire" in an engine stall query).
*   **Diagnosis (Case B - Syntactic Distractor)**: 
    - The Syntactic Distractor ($S_{3b}$) contains the target entity name but in an irrelevant context.
    - **ULL is fooled**: $S_{3b}$ receives high ULL attribution because it primes the target entity tokens.
    - **BERT is NOT fooled**: $S_{3b}$ receives low BERT attribution because the generated response is semantically unrelated to the correct answer.

### 2. Semantic Bias (Similarity/Meaning Bias)
*   **Definition**: The tendency of generation-similarity metrics (like BERTScore or LLM judges) to assign disproportionately high importance to context sentences that mimic the conceptual event, structure, or outcome of the target response, even if they omit the specific facts, links, or target entities required to actually solve the query.
*   **How it Manifestes**: Text embeddings map sentences into dense semantic vector spaces. If a context sentence describes the event of the response using synonyms or category descriptions (e.g., "A twin-engine commercial jetliner caused the runway collision" instead of "Flight Alpha-9 caused..."), the model generates a response that matches the semantic meaning. The BERT embedding similarity is extremely high, prompting the metric to attribute high importance to this sentence, despite the fact that it fails to answer the question (which asks for the *specific* aircraft).
*   **Diagnosis (Case C - Semantic Distractor)**:
    - The Semantic Distractor ($S_{3c}$) describes the event of $R$ but substitutes the target entity with a generic category.
    - **BERT is fooled**: $S_{3c}$ receives high BERT attribution because the generated meaning matches the target response.
    - **ULL is NOT fooled**: $S_{3c}$ receives low ULL attribution because it does not prime the exact target tokens of the response.

## 📋 The 5 Diagnostic Rules

To qualify as a "good benchmark example," a candidate must pass all 5 checks implemented in [`eval_rules.py`](file:///Users/dimitris/Documents/code/python/shapley/biases/eval_rules.py):

| Rule | Area | Condition | Rationale |
| :--- | :--- | :--- | :--- |
| **Rule 1** | Stability | $\forall v \in \mathbf{v}_{\text{normalized}}, \|v\| \le 1.5$ | Prevents extreme, degenerate, or highly unstable Shapley attributions (e.g., large positive/negative cancellations). |
| **Rule 2** | Syntactic | $\text{ULL}(S_{3b}) \ge \max(\text{ULL}(S_1), \text{ULL}(S_2)) - 0.15$ | Verifies that the syntactic distractor successfully triggers syntactic copy-bias under ULL. |
| **Rule 3** | Syntactic | $\text{BERT}(S_{3b}) \le \min(\text{BERT}(S_1), \text{BERT}(S_2)) + 0.15$ | Verifies that semantic-similarity attribution remains immune to syntactic distractor copying (since meaning is irrelevant). |
| **Rule 4** | Semantic | $\text{BERT}(S_{3c}) \ge \max(\text{BERT}(S_1), \text{BERT}(S_2)) - 0.15$ | Verifies that the semantic distractor successfully fools the similarity-based metric. |
| **Rule 5** | Semantic | $\text{ULL}(S_{3c}) \le \min(\text{ULL}(S_1), \text{ULL}(S_2)) + 0.15$ | Verifies that likelihood-based attribution remains immune to the semantic distractor (since exact tokens are missing). |

## 📈 Quality Score Calculation

When an example passes all 5 diagnostic rules, it receives a consolidated quality score. The score is computed as the sum of the **attribution margins** (lower is better for distractors, higher is better for premises):

$$\text{Score} = \text{Margin}_{\text{ULL-B}} + \text{Margin}_{\text{BERT-B}} + \text{Margin}_{\text{BERT-C}} + \text{Margin}_{\text{ULL-C}}$$

Where:
*   $\text{Margin}_{\text{ULL-B}} = \text{ULL}(S_{3b}) - \max(\text{ULL}(S_1), \text{ULL}(S_2))$  *(Measures how strongly $S_{3b}$ fools ULL)*
*   $\text{Margin}_{\text{BERT-B}} = \min(\text{BERT}(S_1), \text{BERT}(S_2)) - \text{BERT}(S_{3b})$  *(Measures how clearly BERT rejects $S_{3b}$)*
*   $\text{Margin}_{\text{BERT-C}} = \text{BERT}(S_{3c}) - \max(\text{BERT}(S_1), \text{BERT}(S_2))$  *(Measures how strongly $S_{3c}$ fools BERT)*
*   $\text{Margin}_{\text{ULL-C}} = \min(\text{ULL}(S_1), \text{ULL}(S_2)) - \text{ULL}(S_{3c})$  *(Measures how clearly ULL rejects $S_{3c}$)*

A higher consolidated score indicates a higher quality benchmark example because it displays a cleaner separation between the logical premises ($S_1, S_2$) and the distractor ($S_3$), while maximizing the targeted biases of interest.
