# LLM Context Attribution Framework

A lightweight, backend-agnostic framework (supporting both Apple Silicon MLX and PyTorch/HuggingFace) for quantifying LLM semantic attribution and sentence interactions (e.g., synergy, redundancy, interference) using cooperative game theory and sparse linear modeling principles.

The framework computes **Shapley Values** (to attribute model generations to specific input context sentences) and **Faith-Shapley Interaction Indices** (to analyze pairwise interactions between context sentences).

---

## Attribution Methods

The framework supports several pluggable attribution methods to score context sentences (configurable via the `method` parameter):

1. **`shapley` (default)**:
   - **Definition**: Computes standard **Shapley Values** (attributing total cooperative value to sentences) and pairwise **Faith-Shapley Interaction Indices** (interaction between sentence pairs).
   - **Computation Cost**: $2^n$ model evaluations.
2. **`banzhaf`**:
   - **Definition**: Computes the **Banzhaf Index** (average marginal contribution of each sentence across all subsets).
   - **Computation Cost**: $2^n$ model evaluations.
3. **`leave_one_out` / `loo`**:
   - **Definition**: Computes **Leave-One-Out** (ablation) attribution: $v(N) - v(N \setminus \{i\})$.
   - **Computation Cost**: $n + 1$ model evaluations (highly efficient for larger contexts).
4. **`individual` / `independent`**:
   - **Definition**: Computes the independent contribution of each sentence: $v(\{i\}) - v(\emptyset)$.
   - **Computation Cost**: $n + 1$ model evaluations.
5. **`context_cite`**:
   - **Definition**: Computes **ContextCite** (NeurIPS 2024) attribution by fitting a sparse linear surrogate model ($y \approx w \cdot X + c$) using LASSO (L1 regularization) or Ridge (L2 regularization) regression over randomly ablated context configurations.
   - **Computation Cost**: Custom sample size $M$ (defaults to $2^n$ if $n \le 5$, otherwise 100). Can be customized by passing an instance of `ContextCiteAttribution`.

---

## Characteristic Value Functions $v(S)$

Each attribution method evaluates a characteristic value function $v(S)$ for a subset (coalition) of sentences $S$. The framework supports several value function formulations (configurable via the `v_type` parameter):

1. **Log-Likelihood Difference (`"logprob_difference"`, default)**:
   - **Formulation**: $v(S) = LL(R \mid Q, S) - LL(R \mid Q, \emptyset)$
   - **Characteristics**: Measures the gain in the likelihood of the response when the context subset $S$ is added. Fast, but can suffer from syntactic copy-bias.
2. **Word-Overlap Discounted LL Difference (`"logprob_wod_difference"`)**:
   - **Formulation**: Applies a penalty to context tokens that overlap with common stopwords and punctuation to mitigate syntactic copy-bias.
3. **Bag-of-Words Baseline Difference (`"logprob_bow_difference"`)**:
   - **Formulation**: Normalizes the log-likelihood by subtracting a baseline log-probability computed using only context-response overlapping words.
4. **Probability Difference (`"probability_difference"`)**:
   - **Formulation**: $v(S) = P(R \mid Q, S) - P(R \mid Q, \emptyset)$
5. **Raw Log-Likelihood / Probability (`"logprob"`, `"probability"`)**:
   - **Formulation**: $v(S) = LL(R \mid Q, S)$ or $P(R \mid Q, S)$ (with distractor contrastive option).
6. **Semantic Similarity (`"similarity_llm_difference"`, `"similarity_bert_difference"`)**:
   - **Formulation**: $v(S) = \text{sim}(R_{\text{grand}}, R_S) - \text{sim}(R_{\text{grand}}, R_{\emptyset})$ where $R_S$ is dynamically generated by the LLM under context $S$, and similarity is measured using either an LLM judge or a BERT encoder (`all-mpnet-base-v2`).

## Codebase Structure

- **`attribution.py`**: The core library entry point. Contains tokenizer and prompt formatting, teacher-forcing log-likelihood calculations, LLM response generation, and the main `compute_attribution` routing function.
- **`methods.py`**: Contains the implementations of all attribution methods (`ShapleyAttribution`, `BanzhafAttribution`, `LeaveOneOutAttribution`, `IndividualAttribution`, `ContextCiteAttribution`) and the `CharacteristicFunction` helper.
- **`backends.py`**: Handles cross-platform model/tokenizer loading, forward pass execution, and log-softmax calculations for both MLX and PyTorch/HuggingFace backends.
- **`example.py`**: A simple example script demonstrating how to load a model and run context attribution using the study room example.


---

## Installation & Setup

The framework is platform-agnostic and supports multiple hardware/inference backends:
- **`mlx`** (default on macOS Apple Silicon): High-performance native inference.
- **`pytorch`** (macOS MPS, CUDA, or CPU): Cross-platform support for Linux, Windows, and macOS.

The backend is auto-detected at runtime, but can be forced if desired (see `backends.py`).

### Setup Instructions

1. Activate your virtual environment:
   ```bash
   source venv/bin/activate
   ```
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
   *(Note: PyTorch (`torch`) is required if using the `pytorch` backend, or when utilizing the `"similarity_bert"` characteristic function.)*

## Core API Reference

### `compute_attribution`
The primary function in `attribution.py`. Computes context attribution using the specified method.

```python
def compute_attribution(
    model, 
    tokenizer, 
    question: str, 
    sentences: list[str], 
    response: str, 
    base_sentences: list[str] = None, 
    distractor: str = None, 
    v_type: str = "logprob_difference", 
    v_func: Callable = None, 
    similarity_fn: Callable = None, 
    max_gen_tokens: int = 128,
    method: str = "shapley"
) -> dict:
```

#### Parameters:
- **`model` / `tokenizer`**: Model and tokenizer loaded via `load_model_and_tokenizer()`.
- **`question`**: The base user question or prompt.
- **`sentences`**: The list of "player" sentences to evaluate. Attributions will be computed over this set.
- **`response`**: The target response text to evaluate. (Can be set to `""` in similarity modes to dynamically generate it via the grand coalition).
- **`base_sentences`**: Background context sentences always included in the prompt.
- **`distractor`**: Used in contrastive formulations: $v(S) = \text{val}(response) - \text{val}(distractor)$.
- **`v_type`**: The formulation type for the characteristic function:
  - `"logprob_difference"` (default): $v(S) = LL(S) - LL(\text{empty})$
  - `"logprob_wod_difference"`: Word-overlap discounted logprob difference (mitigates syntactic copy-bias).
  - `"logprob_bow_difference"`: Bag-of-words baseline normalized logprob difference.
  - `"logprob_unweighted_difference"`: Standard (unweighted) logprob difference.
  - `"probability_difference"`: $v(S) = P(S) - P(\text{empty})$
  - `"logprob"` (raw): $v(S) = LL(S)$
  - `"probability"` (raw): $v(S) = P(S)$
  - `"similarity_llm_difference"`: $v(S) = \text{sim}_{\text{LLM}}(R_{\text{grand}}, R_S) - \text{sim}_{\text{LLM}}(R_{\text{grand}}, R_{\emptyset})$
  - `"similarity_llm"`: $v(S) = \text{sim}_{\text{LLM}}(R_{\text{grand}}, R_S)$
  - `"similarity_bert_difference"`: $v(S) = \text{sim}_{\text{BERT}}(R_{\text{grand}}, R_S) - \text{sim}_{\text{BERT}}(R_{\text{grand}}, R_{\emptyset})$
  - `"similarity_bert"`: $v(S) = \text{sim}_{\text{BERT}}(R_{\text{grand}}, R_S)$
- **`v_func`**: Custom callback `v_func(ll, ll_dist, empty_ll, empty_ll_dist) -> float` for custom mathematical formulations of log probabilities.
- **`similarity_fn`**: Custom callback `similarity_fn(r_grand, r_coalition) -> float` for custom string-similarity scoring.
- **`max_gen_tokens`**: Maximum length limit when dynamically generating responses for similarity evaluations.
- **`method`**: The attribution method to use. Can be a string key (e.g. `"shapley"`, `"banzhaf"`, `"loo"`, `"individual"`, `"context_cite"`), a subclass/instance of `AttributionMethod`, or an object implementing the `attribute(...)` interface.
